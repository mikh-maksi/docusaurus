# Опис та моделювання бізнес-процесів
*"Моделювання процесів. BPMN. Визначення готовності, Визначення завершеності. Формування інструкцій. Метрики проєктного успіху (обсяг, час, бюджет, якість).*
  
  
           	Текстовий опис процесу є найбільш простим для реалізації. Проте, за умови достатньо високого рівня складності проєкту текстовий опис знижує рівнь зрозумілості проєкту.
           	Для зниження рівня когнітивного навантаження, необхідного для розуміння елементів проєкту використовується особливість роботи свідомості людини, що вона сприймає логічну інформацію та образу інформацію за різними каналами.
           	Для реалізації даної можливості для опису бізнес-процесів була розроблена відповідна нотація Business process management notation (BPMN), яка ж, по-суті адптацією підходу формування опису програмних алгоритмів за допомогою блок-схем до опису бізнес-процесів.
           	Для того, щоб спростити опис процесів та збільшити щільність інформації, яку можна передати на одній одиниці квадратної площі були розроблені типи елементів бізнес-процесу: подій, дії, шлюзи із вказанням різних особливостей. А також різні типи з’єднувальних елементів а також інструмент означення ролей за допомогою лінійок.
           	Такий опис має всі ознаки моделі процесу. Адже є її певним спрощенням, яке зберігає основні ключові ознаки реального процесу.
